# P4.1 — `fleet-manager.ts` Split Sketch

**Status**: Draft, awaiting approval before any code changes.
**Branch**: `fix/phase-4-round-3`
**Baseline**: `src/fleet-manager.ts` 2842 行（含 ~436 行純 HTML/JS 字串模板）。

## Goal & Non-Goals

**Goal**：把 `fleet-manager.ts` 切到 ≤ ~1200 行的「協調者」核心，其餘搬到 `Context`-injection pattern 的兄弟模組（與 `instance-lifecycle.ts` / `outbound-handlers.ts` 等既有 split 一致）。

**Non-goals (Round 3)**
- 不重命名既有 public 方法（會擾動 `daemon.ts` 與測試）。
- 不改 `instance-lifecycle.ts` 的 split boundary（已上線、有測試覆蓋）。
- 不調整 IPC / RPC dispatch 行為（P4.2 剛動過）。
- **不抽 Web Dashboard JS 邏輯**（複雜性高、需要瀏覽器 smoke test，留給後續 round）。

## 既有 split 現況（pattern reference）

```
fleet-context.ts            (FleetContext: scheduler/adapter/routing 共用 surface)
instance-lifecycle.ts  680  (LifecycleContext: TmuxManager + daemon spawn)
outbound-handlers.ts   732  (OutboundContext: tool dispatch dictionary)
topic-archiver.ts      122  (ArchiverContext)
statusline-watcher.ts   92  (StatuslineWatcherContext)
agent-endpoint.ts      212  (AgentEndpointContext)
topic-commands.ts      473
routing-engine.ts       53
web-api.ts             691  (WebApiContext)
```

**Pattern**：每個模組宣告一個窄的 `XxxContext` interface，輸出純 function 或 class；`FleetManager implements XxxContext`，呼叫處傳 `this` (或 `this as unknown as XxxContext`)。

## Round 3 抽取候選（共四塊，依風險由低至高）

### ① `src/fleet-dashboard-html.ts` (~440 行) — **lowest risk**

**搬什麼**：`fleet-manager.ts` 2406–2842 整段 `ACTIVITY_VIEWER_HTML` 模板字串。

**機制**：
```ts
// fleet-dashboard-html.ts
export const ACTIVITY_VIEWER_HTML = `<!DOCTYPE html> ...`;
```

**fleet-manager.ts**：把宣告刪掉，加 `import { ACTIVITY_VIEWER_HTML } from "./fleet-dashboard-html.js";`。

**依賴方向**：純常數，零依賴 → 零 circular import 風險。
**測試**：dashboard 是 HTML 字串，沒有單元測試影響；只要 `tsc` 通過＋人工載入 `/activity` 看一次即可。
**淨減行數**：~436 行。

### ② `src/fleet-instructions.ts` (~150 行)

**搬什麼**：
- `INSTRUCTIONS_FILENAME` map（1586–1593）
- `GENERAL_INSTRUCTIONS` 字串常數（1595–1728）
- `ensureGeneralInstructions(workDir, backendName?)` 函式（1731–1740）

**機制**：純 function + 常數匯出，不需 Context。
```ts
// fleet-instructions.ts
export const INSTRUCTIONS_FILENAME: Record<string, string> = { ... };
export const GENERAL_INSTRUCTIONS = `...`;
export function ensureGeneralInstructions(workDir: string, backendName?: string): void { ... }
```

**fleet-manager.ts**：刪掉 static fields + private method，呼叫處改成 named import。
**依賴**：只用到 `node:fs` + `node:path` → 零 circular。
**測試**：純 IO function，現有測試不會崩；可附一個 smoke test 確認檔案產生。
**淨減行數**：~150 行。

### ③ `src/fleet-rpc-handlers.ts` (~290 行) — **medium risk**

**搬什麼**：IPC inbound 與 HTTP 旁支的 CRUD/RPC handler，都是「收到請求 → 操作 scheduler/db → 回 IPC 或 return」：

| 行範圍 | 方法 |
|---|---|
| 986–1027 | `handleScheduleCrud` |
| 1029–1086 | `handleDecisionCrud` |
| 1088–1090 | `resolveDisplayName` |
| 1092–1108 | `handleSetDisplayName` |
| 1110–1128 | `handleSetDescription` |
| 1130–1146 | `handleScheduleCrudHttp` |
| 1148–1188 | `handleDecisionCrudHttp` |
| 1190–1239 | `handleTaskCrudHttp` |
| 1241–1247 | `handleSetDisplayNameHttp` |
| 1249–1255 | `handleSetDescriptionHttp` |
| 1257–1276 | `summarizeToolCall` |
| 1278–1343 | `handleTaskCrud` |

**Context 設計**：
```ts
// fleet-rpc-handlers.ts
export interface RpcHandlersContext {
  readonly scheduler: Scheduler | null;
  readonly fleetConfig: FleetConfig | null;
  readonly logger: Logger;
  readonly instanceIpcClients: Map<string, IpcClient>;
  saveFleetConfig(): void;
  emitSseEvent(event: string, data: unknown): void;
  notifyInstanceTopic(name: string, text: string): void;
  resolveDisplayName(name: string): string;
}

export const rpcHandlers = {
  handleScheduleCrud(ctx: RpcHandlersContext, instanceName: string, msg: ...) { ... },
  handleDecisionCrud(ctx, ...) { ... },
  // ... etc
  handleScheduleCrudHttp(ctx, ...) { ... },
  // ... etc
  summarizeToolCall(tool: string, args: ...) { ... },  // pure
};
```
照 `outboundHandlers` 的格式（dispatch dictionary）。

**FleetManager 改動**：保留 thin wrapper（一行 forwarder）以避免改 `daemon.ts` 與既有測試的 call sites。例如：
```ts
private handleScheduleCrud(name: string, msg: ...) { return rpcHandlers.handleScheduleCrud(this, name, msg); }
```
這些 wrapper 之後 round 4 可以再清；本 round 維持 FleetManager 對外簽章不變。

**依賴方向**：`rpc-handlers → scheduler/types/logger`；不 import fleet-manager → 無 circular。
**測試**：handler 邏輯多走 `scheduler.db`，現有 `tests/scheduler/` + `tests/topic-commands*` 應仍 pass；可選擇加 1 個小單元測試對 `summarizeToolCall`。
**淨減行數**：~290 行（保留 ~12 行 forwarder）。

### ④ `src/fleet-health-server.ts` (~230 行) — **medium-high risk**

**搬什麼**：
- `startHealthServer(port)` — 2156–2381（含 token 寫檔、auth gating、四個 GET endpoint、兩個 POST 與 EADDRINUSE takeover 處理）
- `getUiStatus()` — 2383–2405

**Context 設計**：
```ts
// fleet-health-server.ts
export interface HealthServerContext {
  readonly dataDir: string;
  readonly fleetConfig: FleetConfig | null;
  readonly logger: Logger;
  readonly eventLog: EventLog | null;
  readonly scheduler: Scheduler | null;
  getInstanceDir(name: string): string;
  getInstanceStatus(name: string): "running" | "stopped" | "crashed";
  getSysInfo(): SysInfo;
  lastActivityMs(name: string): number;
  startInstance(name: string, config: InstanceConfig, topicMode: boolean): Promise<void>;
  restartSingleInstance(name: string): Promise<void>;
  emitSseEvent(event: string, data: unknown): void;
}

export function startHealthServer(ctx: HealthServerContext, port: number): {
  server: Server;
  webToken: string;
  getUiStatus: () => unknown;
};
```

**FleetManager 改動**：
- `private healthServer / private webToken / private startedAt` → 由 `startHealthServer` 回傳並存到 field（或回傳一個 `{ server, getStatus, dispose }` object）。
- 保留 `getUiStatus()` thin wrapper 給 `emitSseEvent` 呼叫處。

**Risk note**：兩個 `(req, res) => { handleAgentRequest(req, res, this as ... AgentEndpointContext); }` / `handleWebRequest(req, res, url, this as ... WebApiContext)` 的「`this` 借閱」要繼續成立——把 `ctx` 傳下去就好（`ctx` 已是 `HealthServerContext` superset of 它們的需求 by virtue of FleetManager 實作多介面）。

**依賴方向**：`fleet-health-server → web-api / agent-endpoint`（已存在的 import 路徑，不引入新 cycle）。
**測試**：手動 smoke：`/health` 200、`/api/fleet` 帶 token 200、無 token 401、`/restart/:name` 路徑。沒有自動測試覆蓋這層；split 不增刪行為。
**淨減行數**：~230 行。

## Dependency Graph 檢查（避免 circular）

```
fleet-dashboard-html.ts   →  (none)
fleet-instructions.ts     →  node:fs, node:path
fleet-rpc-handlers.ts     →  scheduler/, types, logger, channel/ipc-bridge
fleet-health-server.ts    →  web-api.ts, agent-endpoint.ts, fleet-context.ts
                           ↑
                  fleet-manager.ts
                           ↓ (implements)
              FleetContext / Lifecycle / Archiver / StatuslineWatcher /
              Outbound / AgentEndpoint  + (new) RpcHandlersContext / HealthServerContext
```

新模組**不**反向 import `fleet-manager.ts` —— 與現存四個 extracted 模組一致。`web-api.ts` 與 `agent-endpoint.ts` 既有 contract 不變。
